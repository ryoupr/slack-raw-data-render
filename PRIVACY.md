# Privacy Policy for Slack Markdown Renderer

**Last updated: February 10, 2026**

## Overview

Slack Markdown Renderer is a Chrome extension that renders Markdown content in Slack RAW file pages. This privacy policy explains how we handle data when you use our extension.

## Data Collection and Usage

### What Data We Access
- **Website Content**: The extension accesses Markdown text content from Slack RAW file pages (https://files.slack.com/files-pri/*) that you actively visit.

### How We Use This Data
- **Local Processing Only**: All Markdown content is processed locally within your browser to render formatted output.
- **No Data Storage**: We do not store, save, or retain any of your data.
- **No External Transmission**: Your data is never sent to external servers or third parties.

### What We Don't Collect
- Personal identifying information
- Authentication credentials
- Financial information
- Health information
- Location data
- Web browsing history outside of active Slack file pages
- User activity tracking

## Data Security

- All processing occurs locally in your browser
- No data is transmitted over the internet
- No data is stored on our servers (we don't have any servers)
- The extension only operates on pages you actively visit

## Permissions Explanation

### Host Permission (https://files.slack.com/files-pri/*)
- **Purpose**: Enables automatic activation on Slack RAW file pages
- **Scope**: Limited exclusively to Slack's file sharing domain
- **Usage**: Necessary for the extension's core functionality

## Third-Party Libraries

This extension uses the following open-source libraries, all included in the extension package:
- **Marked.js**: For Markdown parsing (no data transmission)
- **Prism.js**: For syntax highlighting (no data transmission)

## Data Sharing

We do not:
- Sell your data to third parties
- Share your data with advertisers
- Use your data for purposes unrelated to Markdown rendering
- Transmit your data outside your local browser
